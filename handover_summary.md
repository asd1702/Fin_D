# 프로젝트 현황 및 이슈 분석 보고서

이 문서는 현재 **"Fin:D 금융 인텔리전스 대시보드"** 프로젝트의 개발 진행 상황, 구현된 기능, 그리고 최근 발생한 **사용자 DB 데이터 연동 문제**에 대한 상세 분석을 담고 있습니다. 다른 LLM이나 개발자가 컨텍스트를 이어받을 때 참조하십시오.

## 1. 주요 요구사항 및 구현 기능

사용자는 대시보드(`Dashboard`) 페이지의 기능 개선과 사용자 경험(UX) 강화를 요청했습니다.

### A. 뉴스 피드 섹션 (`NewsSection`)
*   **요구사항**: 사용자가 즐겨찾기한 기업들의 최신 뉴스를 대시보드 최상단에 노출.
*   **구현 상세**:
    *   **백엔드**: `GET /news/favorites` 엔드포인트를 구현. 단순 DB 조회뿐만 아니라, **"On-Demand Fetching"**을 적용하여 DB에 최신 뉴스(최근 7일)가 없으면 **FMP API**를 실시간으로 호출하여 데이터를 업데이트하도록 로직 강화.
    *   **프론트엔드**: `NewsSection.tsx` 구현. 데이터 로딩 상태 처리 및 "뉴스가 없을 경우"에 대한 빈 상태 UI(Empty State) 추가.
    *   **이슈 해결**: 로컬 타임존 차이로 인한 데이터 누락 가능성을 배제하기 위해 필터링 기준을 48시간에서 **7일**로 완화.

### B. 관심 기업 리스트 카러셀 (`FavoriteCompanies`)
*   **요구사항**: 뉴스 섹션과 캘린더 사이에, 즐겨찾기한 기업 정보를 카드 형태로 **가로 스크롤(좌우 슬라이드)**이 가능하게 배치.
*   **구현 상세**:
    *   **컴포넌트**: `FavoriteCompanies.tsx` 신규 개발.
    *   **기능**: `GET /user/favorites` API를 통해 즐겨찾기 종목(`ticker`)을 가져온 뒤, 각 종목의 상세 프로필(`CompanyProfile`)과 현재 주가(`Quote`)를 병렬로 호출하여 `CompanyCard` 컴포넌트로 렌더링.
    *   **UI/UX**: Flexbox 기반의 가로 스크롤 컨테이너(`overflow-x: auto`)를 적용하여 모바일 및 데스크탑에서 자연스러운 카드 탐색 경험 제공.

## 2. 사용자 DB 연결 문제 및 해결 분석

최근 개발 과정에서 **"로그인한 사용자의 즐겨찾기 데이터가 프론트엔드에서 `undefined`로 인식되는 치명적인 문제"**가 발생하여 집중 디버깅을 수행했습니다.

### A. 문제 현상 (Symptom)
*   사용자가 로그인을 완료했음에도 대시보드의 뉴스 피드와 관심 기업 리스트가 비어 있거나 로딩되지 않음.
*   프론트엔드 콘솔 및 서버 로그에 다음과 같은 404 에러가 다수 발생:
    > `GET /api/v1/company/quote/undefined 404 Not Found`
    > `[Cache MISS] FMP API 호출: /profile/undefined`
*   **원인 추정**: 프론트엔드가 백엔드로부터 받은 즐겨찾기 리스트 안의 `ticker` 값이 `undefined`이거나 비어 있어서, 후속 API 호출 시 잘못된 URL을 요청함.

### B. 원인 상세 분석 (Root Cause)
*   **Pydantic 버전 호환성 문제**:
    *   현재 서버 환경에는 최신 버전인 **Pydantic V2 (v2.12.3)**가 설치되어 있음.
    *   그러나 백엔드 코드(`schemas.py`)는 Pydantic V1 문법인 `orm_mode = True`를 사용하고 있었음.
    *   Pydantic V2에서는 `orm_mode`가 `deprecated` 되었고, SQLAlchemy 객체(ORM 모델)를 Pydantic 모델로 변환(직렬화)할 때 V1 방식이 제대로 작동하지 않아 **빈 JSON 객체 `{}` 또는 필드가 누락된 객체**가 반환됨.
    *   이로 인해 DB에는 데이터가(`ticker="AAPL"`) 정상적으로 존재함에도, API 응답 JSON에는 `ticker` 필드가 없거나 무시되어 프론트엔드로 전송됨.

### C. 해결 조치 (Resolution)
1.  **스키마 마이그레이션 (V1 -> V2)**:
    *   `app/schemas.py` 내 모든 Pydantic 모델의 `Config` 클래스 설정을 수정.
    *   `orm_mode = True` (V1) ➔ **`from_attributes = True` (V2)** 로 변경.
2.  **직렬화 테스트 검증**:
    *   `debug_serialization.py` 스크립트를 작성하여 실제 DB의 SQLAlchemy 객체가 Pydantic V2 어댑터(`TypeAdapter`)를 통해 정상적으로 JSON 변환됨을 확인 (`{"ticker": "AAPL", ...}` 출력 성공).
3.  **방어 코드 추가**:
    *   **백엔드**: `user_data.py`에 로깅을 추가하여 요청된 즐겨찾기 데이터의 실제 내용을 서버 콘솔에 출력하도록 조치.
    *   **프론트엔드**: `FavoriteCompanies.tsx`에서 `ticker`가 없는 잘못된 데이터가 오더라도 무조건 API를 호출하지 않고, `console.warn` 후 스킵하도록 하여 **불필요한 404 에러 방지**.

### D. 현재 상태 (Current Status)
*   백엔드 데이터 직렬화 문제는 **해결됨**.
*   사용자 `7`번(테스트 계정)의 데이터가 정상적으로 API를 통해 제공될 준비가 됨.
*   현재 프론트엔드에서 새로고침 시 즐겨찾기 목록과 뉴스 피드가 정상 표출되어야 함.

---

**요약**: "DB 연결이 안 된 것"처럼 보였던 이유는 DB 접속 실패가 아니라, **DB에서 꺼낸 데이터를 API 응답(JSON)으로 변환하는 과정에서 라이브러리 버전 차이로 인해 데이터가 유실되었기 때문**입니다. 현재는 해당 호환성 문제를 수정 완료했습니다.
